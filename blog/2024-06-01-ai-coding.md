## ✨ GitHub Copilot 技巧

### 1. 写好注释引导

```javascript
// 创建一个防抖函数，接收函数和延迟时间，返回防抖后的函数
function debounce(fn, delay) {
  // Copilot 会根据注释生成代码
}

// 发送 POST 请求到 /api/users，携带 JSON 数据
async function createUser(userData) {
  // ...
}
```

### 2. 提供上下文

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  createdAt: Date;
}

// Copilot 会根据接口定义生成匹配的代码
function formatUser(user: User): string {
  // ...
}
```

### 3. 使用示例引导

```javascript
// 输入示例，帮助 Copilot 理解模式
const users = [
  { name: 'John', age: 25 },
  { name: 'Jane', age: 30 }
];

// 按年龄排序
const sortedUsers = // Copilot 会补全
```

### 4. 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Tab` | 接受建议 |
| `Esc` | 拒绝建议 |
| `Alt+]` | 下一个建议 |
| `Alt+[` | 上一个建议 |
| `Cmd+Enter` | 打开建议面板 |

## 🎯 Cursor 技巧

### 1. Cmd+K 快速编辑

```javascript
// 选中代码，按 Cmd+K，输入指令
// "添加错误处理"
// "转换为 TypeScript"
// "优化性能"
```

### 2. Cmd+L 对话模式

```markdown
对话示例：
- "解释这段代码的作用"
- "帮我写一个单元测试"
- "这段代码有什么问题"
- "如何重构这个函数"
```

### 3. @ 引用

```markdown
@file:src/utils.ts 引用文件
@folder:src/components 引用文件夹
@web 搜索网络
@docs 引用文档
```

### 4. Composer 功能

```markdown
用于复杂任务：
- "创建一个登录页面，包含表单验证"
- "实现一个 TODO 应用，使用 React + TypeScript"
- "给这个项目添加单元测试"
```

## 📝 Prompt 技巧

### 1. 明确具体

```markdown
# ❌ 模糊
"写一个函数"

# ✅ 具体
"用 TypeScript 写一个函数，接收日期字符串（格式：YYYY-MM-DD），
返回该日期是星期几（中文）"
```

### 2. 分步骤

```markdown
# ❌ 一次性要求太多
"写一个完整的用户管理系统"

# ✅ 分步骤
1. "设计用户数据模型"
2. "实现用户 CRUD API"
3. "添加认证中间件"
4. "编写单元测试"
```

### 3. 提供上下文

```markdown
# ✅ 提供项目背景
"我在使用 Next.js 14 + App Router + Prisma，
需要实现一个用户列表页面，支持分页和搜索"
```

### 4. 指定格式

```markdown
# ✅ 明确输出格式
"用 Markdown 表格列出 React 18 的新特性"
"用代码块展示示例，并添加注释"
```

## ⚠️ 注意事项

### 1. 验证代码

```javascript
// AI 生成的代码可能有 bug
// 务必测试和 review

// ❌ 直接使用
// ✅ 理解 → 测试 → 使用
```

### 2. 安全意识

```javascript
// ❌ 让 AI 处理敏感信息
"帮我加密这个 API Key: sk-xxxx..."

// ✅ 使用占位符
"帮我写一个加密函数，使用环境变量中的 API Key"
```

### 3. 隐私考虑

```markdown
- 代码可能上传到云端
- 不要发送公司机密代码
- 遵守公司 AI 使用政策
```

### 4. 保持学习

```markdown
- AI 是助手，不是替代品
- 理解代码原理，不只是复制粘贴
- 持续学习新技术
```

## 🎨 实用场景

### 1. 快速原型

```markdown
"用 React 创建一个天气应用的 UI 原型，
包含搜索框、当前天气卡片、7天预报列表"
```

### 2. 代码解释

```markdown
"解释这段正则表达式的含义：
/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d]{8,}$/"
```

### 3. Bug 调试

```markdown
"这段代码运行时报错：Cannot read property 'x' of undefined
帮我分析可能的原因和解决方案"
```

### 4. 代码转换

```markdown
"把这个 JavaScript 函数转换为 TypeScript，添加类型定义"
"把这个 Class 组件转换为函数组件 + Hooks"
```

### 5. 单元测试

```markdown
"为这个函数编写 Jest 单元测试，覆盖正常情况和边界情况"
```

### 6. 文档生成

```markdown
"为这个 API 生成 JSDoc 注释"
"生成 README 文档，包含安装、使用、API 说明"
```
